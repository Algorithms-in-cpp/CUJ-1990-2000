Intuitive Edge Launches Datatypes++ 1.0

Intuitive Edge Corporation has released Datatypes++ 1.0, a Windows class library that brings datatypes to C++. Datatypes++ 1.0 offers 25 text, numeric, time, and binary datatypes similar to those found in Java, SQL, Visual Basic, ODBC, Delphi, MFC, and C++ STL.

The text types can be converted to and from all other types in dozens of dialects and include Unicode support. The numeric types include a decimal type that can represent fixed or dynamic high-precision numbers. The time types (time, date, and timestamp) can represent time in 100 world clocks with nanosecond precision. The binary datatypes include support for binary large objects and bit sets.

Datatypes++ 1.0 costs $298 for an individual license and includes support for Microsoft and Borland C++ compilers.

For more information, contact Intuitive Edge Corporation, 800.586.4090 or +1.425.746.5925; Info@IntuitiveEdge.com; www.IntuitiveEdge.com.

Intel Introduces VTune Performance Analyzer v4.5, Intel C/C++ Compiler v4.5, and Intel Fortran Compiler v4.5

Intel Corporation has released VTune Performance Analyzer v4.5, Intel C/C++ Compiler v4.5, and Intel Fortran Compiler v4.5, software development tools for the latest Intel CPUs. The VTune Performance Analyzer v4.5 helps programmers find and fix application bottlenecks that may limit performance. It can profile both JIT-compiled code and native Windows applications. These compilers also feature intrinsics and class libraries that support the Intel processor instructions, eliminating the need to program directly in assembly language.

The VTune Performance Analyzer v4.5 costs $429, the C/C++ Compiler v4.5 costs $299, and the Fortran Compiler v4.5 costs $349.

XML Software Announces InterAccess v1.1

XML Software Corporation has announced InterAccess v1.1, a product that provides access to any ODBC/OLEDB-compliant SQL database via the Internet. InterAccess v1.1 implements a client/server architecture much like FTP and uses an XML protocol to provide access to SQL databases that have ODBC or OLEDB drivers. It consists of three components: InterAccess Server, the InterAccess client COM DLL, and the InterAccess Browser.

The COM DLL provides COM objects and interfaces ready to drop into Visual Basic, C++, or other RAD tool programs. The COM object model is similar to the Microsoft ADO object model.
