Willows Releases Willows Toolkit 2.0

Willows Software, Inc. has released the 32-bit version of the Willows Toolkit, with support for Borland International's ObjectWindows Library (OWL), designed to create a seamless migration path from Windows applications to other platforms. The library is based on a single set of source code that can be compiled and linked for the target environment, allowing developers to deliver cross-platform, full-featured applications that run at native execution speed. Because the application code remains compatible with Windows, developers can exploit non-Windows platforms with their Windows applications. Unique features let developers combine source code and existing Intel binary objects (i.e., DLLs) to take advantage of third-party component technology without requiring vendor source code. The Willows Toolkit has no ties to Windows source code and no run time royalties.

The Willows Toolkit now offers OWL encapsulations of the Windows 95 common controls such as Tab Controls, Property Pages, and Tool Tips, as well as encapsulations of splitter windows, splash screens, dockable toolbars, print preview, and animation. Additionally, the Willows Toolkit supports synchronization objects for multi-threaded development.

Pricing for all supported UNIX platforms (Sun Solaris, SunOS, IBM AIX, HP-UX, Digital UNIX, SGI IRIX, SCO UNIX, UnixWare, and Linux) is $4,995 per developer. A Linux version with no source code is also available for $1,250 per developer. Pricing for the Macintosh is $495 per developer. Educational discounts are available.

IONA Releases OrbixTalk CORBA Events Services

IONA Technologies has released OrbixTalk, its first implementation of the CORBA Events Services. OrbixTalk is a scalable messaging system that implements an events paradigm, which enables reliable decoupled messaging between objects and applications in a distributed system.

OrbixTalk works seamlessly with IONA's Orbix object request broker as well as with other CORBA services provided by IONA Technologies. Scalability is ensured by the asynchronous nature of the OrbixTalk architecture and the underlying UDP/IP (User Datagram Protocol/Internet Protocol) multicast protocol.

In addition to the core OrbixTalk product, a MessageStore component provides persistence, ensuring messages are delivered in the event of client or server failure. MessageStore uses traditional store-and-forward mode, a store-only mode, and on-demand playback.

OrbixTalk costs $1,500 per developer seat on UNIX, $750 on Windows NT. MessageStore pricing is available on request.

Walnut Creek Releases FreeBSD v2.1

The FreeBSD Project, in association with Walnut Creek CDROM, has released FreeBSD version 2.1, a full 4.4 BSD Lite-based UNIX-like operating system for the PC. FreeBSD comes with a full development environment with C, C++, and FORTRAN compilers and debuggers as standard. Full source code provided for the entire operating system and all utilities make a truly open system. FreeBSD installs easily alongside other operating systems such as DOS and Windows (3.1, 95, or NT) and can share files directly with DOS, both on the local disk and over the network with SAMBA and PCNFS.

The largest change to the FreeBSD operating system since the last release, 2.0, is a revamped VM system with a merged VNVfile buffer cache that increases performance and reduces FreeBSD's memory footprint, making a 5MB configuration a more acceptable minimum. Other enhancements include full NIS client and server support, transaction TCP support, dial-on-demand PPP, an improved SCSI subsystem, early ISDN support, support for FDDI and Fast Ethernet (100Mbit) adapters, and improved support for the Adaptec 2940 (WIDE and narrow) and 3940 SCSI adaptors, and many hundreds of bug fixes.

FreeBSD is available free of charge on the Internet at ftp.freebsd.org and in a convenient CD-ROM distribution from Walnut Creek CDROM for $39.95. Walnut Creek CDROM's 1200-user Internet site at ftp.cdrom.com uses a server running FreeBSD 2.1.

BetterState PRO Generates Multithreaded C++ Code

R-Active's BetterState PRO Version 4.0, a PC-based design tool for statecharts, now generates C++ code that supports true preemptive multithreading for Windows 95 and Windows NT applications. Earlier versions of BetterState realized concurrency using a single thread of computation in which a compile-time, non-preemptive schedule of state transitions was used to implement concurrecy. However, many real-time and user interface applications require true preemptive multithreading capabilities. With the new code generator, the underlying Windows 95 or Windows NT operating system is used to realize a true, preemptive, multithreaded application. Under Windows NT, such code can also be used for a true multiprocessor system.
